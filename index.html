Have you ever asked…

Who you gonna call?

When something goes down, who are you supposed to contact? In-house I.T.? Microsoft? 911?

Why must we use Internet Explorer? or Why is this Web Page a file?

This is because sometimes organizations want to allow local files to be opened from a Network File store in web browser. However, without going through http or WebDAV protocol, the only options for linking to local files are:

  • Internet Explorer with lowered security settings (websites hosted via HTTP)
  • Chrome with --allow-access-from-files argument (website hosted locally as .html file)
  • Firefox (website hosted locally as .html file)

Where is the documentation?

Documentation is occasionally housed in a different location than the Application. Even when you can find the documentation from the application, sometimes it involves a deal of navigation or it’s the product vendor’s documentation and not your department’s.

How do I find…

You can slowly learn where things are and then bookmark and create shortcuts. But static pointers tend to break when processes, technology, people, and organizations change.

The answer is here

With Wan der Vaal, you can search for, filter, and sort your information systems based on Metadata

Wan der Vaal can be installed as a program on users computers. A self-updater allows the application to get updated with fresh code, and the Information System data can be fed in from a Database or Rest API endpoint.

File:// and http(s):// are supported as well as any other protocols the base filesystem recognizes. Web links will be opened either in-application or using user’s default web browser, File links will be handled by the base Operating System.

Windows and Linux Supported.
